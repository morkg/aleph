# aleph

`@morkg/aleph` is a lightweight library for generating unique, time-based tokens using AES-256-CBC and UUIDv7. The package exposes two main functions: `encode` to create encrypted tokens and `decode` to retrieve the original content, with optional expiration support.

## Installation

Install with npm or yarn:

```bash
npm install @morkg/aleph
```

or:

```bash
yarn add @morkg/aleph
```

## Importing

### With ES modules

```ts
import aleph, { encode, decode, origin } from "@morkg/aleph";
```

### With CommonJS

```js
const aleph = require("@morkg/aleph").default;
const { encode, decode, origin } = require("@morkg/aleph");
```

## Usage

### encode(text, secret, expire?)

Generates a unique encrypted token that embeds the original text.

- `text`: the string or payload to protect.
- `secret`: master key used to derive the AES-256 key.
- `expire` (optional): `Date` or `number` in milliseconds representing when the token should expire.

Returns a long hexadecimal string containing:
- a randomly generated UUIDv7,
- the original text encrypted with that UUID,
- and the UUID encrypted with the key derived from the secret.

#### Example

```ts
import { encode } from "@morkg/aleph";

const secret = "my-secret-password";
const payload = "user:12345";
const token = encode(payload, secret);
console.log(token);
```

#### Example with expiration

```ts
import { encode } from "@morkg/aleph";

const secret = "my-secret-password";
const payload = "access:admin";
const expiresAt = new Date(Date.now() + 60 * 60 * 1000); // expires in 1 hour
const tokenWithExpiry = encode(payload, secret, expiresAt);
console.log(tokenWithExpiry);
```

### decode(token, secret)

Retrieves the original text from a token created by `encode`.

- `token`: the hexadecimal token produced by `encode`.
- `secret`: the same secret used when the token was created.

The function decrypts in two stages:
1. decrypt the initial UUID using the master secret;
2. use that UUID to decrypt the payload and return the original value.

If the token contains an expiration timestamp and it has already passed, the function throws a `Token expired` error.

#### Example

```ts
import { decode } from "@morkg/aleph";

const secret = "my-secret-password";
const token = "..."; // token generated by encode

try {
  const original = decode(token, secret);
  console.log("Original content:", original);
} catch (error) {
  console.error("Failed to decode token:", error.message);
}
```

### origin(token, secret)

Extracts the exact creation date of an Aleph token.

- `token`: the hexadecimal token produced by `encode`.
- `secret`: the same master key used during encryption.

The function decrypts only the first part of the token to reveal the internal UUIDv7, extracts the embedded timestamp bits, converts them from hexadecimal to milliseconds and returns a `Date` object representing when the token was created.

#### Example

```ts
import { origin } from "@morkg/aleph";

const token = "..."; // token generated by encode
const birthDate = origin(token, "my-secret-password");
console.log(birthDate.toISOString());
```

## How it works

The library uses:

- `crypto.createHash("sha256")` to derive a 32-byte key from the secret.
- `crypto.createCipheriv("aes-256-cbc")` to encrypt text as hex.
- `uuidv7()` to generate a time-based UUID used as an internal encryption key.
- an internal format that concatenates the initialization vector (IV) with the encrypted payload.

### Notes

- `encode` always returns a new and unique token, even for the same text and secret.
- `decode` requires the same `secret` that was used by `encode`.
- If the token has expired, `decode` throws an error and does not return the original text.
